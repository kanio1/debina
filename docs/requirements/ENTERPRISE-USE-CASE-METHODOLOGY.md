# Enterprise Use-Case Methodology

Use-Case Foundation is Debina's universal base: an external primary actor pursues a goal across a declared system boundary through a readable set of successful, challenged and failure flows. Cockburn structures a narrative when the selected profile needs it; Use-Case 2.0 connects selected behavior to realization and tests; Example Mapping records discovery rather than claimed consensus. The method responsibility boundaries are authoritative in [the methodology assurance map](../governance/methodology-assurance/METHOD-RESPONSIBILITY-MAP.md).

## Adaptive elaboration

Choose one explicit `detail_profile`:

- `OUTLINE` — discovery or source-blocked candidate; never READY.
- `ESSENTIAL` — normal implementation preparation with guarantees, flows, rules, sources, slices, tests, quality and architecture links.
- `FULLY_DRESSED` — justified for money movement, finality, security boundaries, interoperability, complex recovery/concurrency or an architecture-significant trade-off. It adds stakeholders, trigger, detailed anchored extensions, variations and relevant volume assumptions.

`FULLY_DRESSED` is not a badge for every ENFORCED use case. The templates are [outline](USE-CASE-OUTLINE-TEMPLATE.md), [essential](USE-CASE-ESSENTIAL-TEMPLATE.md), and [fully dressed](USE-CASE-FULLY-DRESSED-TEMPLATE.md); fields have one shared definition in [USE-CASE-METHOD.md](USE-CASE-METHOD.md).

## Required metadata and honesty

Every new or materially revised ENFORCED use case declares `system_of_interest`, `system_boundary`, `primary_actor_type`, `actor_goal`, `goal_level`, `detail_profile`, discovery status, source-evidence status and architecture-evaluation type. Internal Debina modules are architecture realization, not actors. AI-authored discovery starts `AI_DRAFT` / `NOT_REVIEWED`; it cannot imply stakeholder agreement. Material open questions block readiness.

Use `ATAM_INSPIRED_DESK_REVIEW` for desk analysis unless durable workshop evidence supports a stronger classification. ISO/EPC/rail facts require source evidence appropriate to the claim; current code only demonstrates current Debina behavior. Shared terminology remains authoritative in the Phase B catalogues.
