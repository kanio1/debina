## Environment gate result

The installed CLI/Engine and Go toolchain work, but the rootful Podman socket cannot be used by the current user. The exact read-only probe was:

```text
podman --remote --url unix:///run/podman/podman.sock ps
→ dial unix /run/podman/podman.sock: connect: permission denied
```

Consequently Dagger auto-detects the rootless Podman endpoint instead. It can start Engine `v0.21.4`, but the observed privileged Engine has a PID limit of 2048. This does not satisfy the documented rootful-Podman safety gate, so no repository module has been generated.

## Required user actions before retry

1. Authorize this user to connect to `/run/podman/podman.sock` through the workstation's approved rootful-Podman access policy, then explicitly confirm that authorization. Access to a rootful Podman API is security-sensitive because it permits host-privileged container management.
2. Set a safe Dagger Engine PID limit through the approved host policy and explicitly confirm that change. Do not use a blanket unlimited value without the workstation owner's decision.
3. Retry with `podman --remote --url unix:///run/podman/podman.sock ps`, then `dagger version && dagger core version && dagger check --help`. The first repository-independent Dagger operation must be the deterministic container probe.
